# Fix Mobile Chess Board Distortion - Plan

## Problem
On mobile screens with limited vertical space, the chess board squares become rectangles instead of squares. The `aspect-ratio: 1/1` constraint on the board is being overridden by the grid container's row sizing.

## Root Cause
In `/styles/layout.css:76`, the `#board-container` uses `grid-template-rows: 1fr auto`. This means:
- The first row (board grid) gets "remaining space" after accounting for the second row (file labels)
- When vertical space is limited, `1fr` forces the board to compress vertically
- This breaks the `aspect-ratio: 1/1` constraint on `.chess-board-grid`

## Solution
Change the board container grid rows to prioritize maintaining the board's aspect ratio. Instead of `1fr auto`, use `auto auto` for the rows:

### Changes needed:
1. In `/styles/layout.css:76`:
   - Change `grid-template-rows: 1fr auto;` to `grid-template-rows: auto auto;`
   
   This ensures:
   - The board gets exactly the height it needs (maintaining aspect ratio)
   - File labels get exactly the height they need
   - No vertical compression of the board occurs

## Verification
After applying the fix:
- Chess board will always maintain perfect square aspect ratio on all screen sizes
- Squares will always be perfectly square
- No distortion even on very small mobile screens
- File labels will still display correctly below the board

## Additional Improvements (Optional)
To make the fix even more robust, we can add `align-self: start` to the board grid to prevent any stretching:
- Add `align-self: start;` to `.chess-board-grid` styles
